## 一、背景

v3 三角流程裡,「老師核准計畫書、工程師才准動手」這道 gate 目前是**社會性**的——靠老師人工批改、甲方拍板。代價是每次都要人在迴路。

`Ancienttwo/repo-harness`（MIT）示範把這道 gate **機械化**:用 Claude Code 的 PreToolUse hook,在工程師真要寫 production code 時,依「檔案路徑 × plan 狀態」做**確定性**判斷,未核准就 `exit 2` 硬擋。同時用 Stop hook 把 session 交接狀態以 **git diff 機械算出**寫進固定檔,下個 session 開場自動注入——零 LLM、零 token。

本工單**不採用** repo-harness 本體,只移植三個經驗證的機制,重寫為我們自己可控、可審計的最小實作。

---

## 二、設計原則（不可違背）

1. **Gate 不猜意圖**。判斷只依「動到的檔案路徑」+「plan 檔狀態」兩個確定性輸入。**禁止**用自然語言 intent 分類決定要不要擋。
2. **交接狀態機械算,不靠模型生成**。next-action、changed files 一律從 `git` 與 repo 檔案狀態算出,hook 內**不得呼叫任何 LLM**。
3. **三檔開關,預設最輕**。每道 guard 支援 `enforce | advice | off`,吃環境變數,預設 `advice`（只提醒不擋）。確認順手後才由甲方手動切 `enforce`。
4. **退出碼語意嚴格遵守**。擋 = `exit 2`（且訊息寫 stderr）;放行 = `exit 0`;**絕不用 `exit 1` 當擋**（Claude Code 視 `exit 1` 為 non-blocking,動作照樣執行,等於沒擋）。
5. **可移植、純 bash**。除 `git`、`jq`（optional,須有 awk fallback）外不引入新依賴。InvestSys、NutriPilot 都要能直接落地。
6. **最小侵入**。hook 掛在**專案級** `.claude/settings.json`（repo 內,可進 git）,**不動** user-level `~/.claude/settings.json`。

## 三、交付項目

### D1. `hooks/lib/workflow-state.sh`（共用函式庫）

- `get_active_plan()`：讀 marker 檔 `.ai/harness/active-plan`（內容為 plan 檔相對路徑）。marker 不存在或指向的檔不存在 → return 1。
- `get_plan_status(plan_file)`：從 plan 檔抓 `**Status**:` 欄位值（參考:`awk '/\*\*Status\*\*:/ {sub(/^.*\*\*Status\*\*: */,""); print; exit}'`）。
- `set_active_plan(plan_file)` / `clear_active_plan()`：寫/清 marker。寫入時同時記下 `pwd -P`,供 worktree 一致性檢查（見第四節 Q4）。
- `contract_allows_path(contract_file, file_path)`：解析 contract 檔裡 ```yaml 區塊的 `allowed_paths:` 清單,逐項比對。規則:項目以 `/` 結尾 → 前綴比對;否則 → glob 比對。任一命中 return 0,否則 return 1。
- `policy_get(json_path, default)`：從 `.ai/harness/policy.json` 讀設定,jq 優先、awk fallback、檔不存在回 default。

### D2. `hooks/pre-edit-guard.sh`（PreToolUse on `Edit|Write`）

1. 從 stdin JSON 取 `file_path`（依序試 `.tool_input.file_path` → `.file_path`,jq 解析,awk fallback）。取不到 → `exit 0`。
2. **workflow surface 放行**:路徑符合 `plans/ tasks/ docs/ .ai/ .claude/ *.md` → `exit 0`。
3. **絕對路徑或 repo 外**放行（只管 repo-scoped 相對路徑）。
4. 讀 gate mode（env `V3_EDIT_PLAN_GATE` > `policy.guards.edit_plan_gate` > 預設 `advice`）。`off` → `exit 0`。
5. 依序跑:
   - **PlanStatusGuard**：`get_active_plan` 無 active plan,或 plan 狀態 ∈ {`Draft`, `Annotating`}（未核准）→ 擋。
   - **ContractScopeGuard**（若該 repo 啟用 contract）：`contract_allows_path` 不通過 → 擋。
   - 擋的處理:`enforce` → 寫 stderr + `exit 2`;`advice` → 寫 stdout 提醒 + `exit 0`。
6. （加分）TDD 提醒:改 `.ts/.py/.swift` 但找不到對應測試檔 → stdout 提醒,**不擋**。

### D3. `hooks/lib/write-handoff.sh`（被 Stop hook 呼叫）

`write_handoff(reason)`,從 git 機械算出並寫入 `.ai/harness/handoff/resume.md`:active plan 路徑與狀態;changed files（`git diff --name-only HEAD` ∪ `git ls-files --others --exclude-standard`,去重排序,>80 行截斷）;`git diff --shortstat HEAD`;產生時間、reason。**不呼叫 LLM**。非 git repo 降級為只寫時間+reason。

### D4. `hooks/stop-orchestrator.sh`（Stop hook）

1. 無條件 `write_handoff "session-stop"`。
2. （加分）PlanCompletenessGate:偵測本 session 是否產出「像 plan 的長訊息」（≥240 字元 + 命中 `P1|P2|P3|方案|計畫|success criteria|rollback`）但未落檔,signature 去重,擋時輸出 `{"decision":"block","reason":...}` 並附「計畫書需回答的問題」清單。

### D5. `hooks/session-start-context.sh`（SessionStart hook）

讀 `resume.md` + `tasks/current.md`（若存在）,stdout 純文字（`exit 0`）輸出。**關鍵**:明標 **recovery context only**——若使用者本輪訊息含實際附件/檔案路徑,優先讀使用者輸入,別讓舊狀態蓋過當前任務。

### D6. `.claude/settings.json` 範本 + `INSTALL.md`

- 範本掛 hook（SessionStart / PreToolUse:`Edit|Write` / Stop）,每個 hook 用 `git rev-parse --show-toplevel` 解析 repo root、`timeout: 30`,matcher 嚴格寫 `Edit|Write`（大小寫精確）。
- `INSTALL.md`:三檔開關怎麼切、如何先在 InvestSys 用 `advice` 試跑、確認後切 `enforce`、一鍵停用。

# v3-003 — 版控稽核：補齊未追蹤原始碼與測試

**開單日期**：2026-06-13
**類型**：維護（非 TDD）
**執行者**：頂層 Sonnet（不走 engineer-tdd）
**狀態**：HITL gate 等待核可

---

## 目標

把 InvestSys 磁碟上已存在、但從未進入 git 版控的原始碼與測試補齊追蹤，並對非必要檔案補 .gitignore 規則。

**範圍界定**：`git status --porcelain` 輸出的 `??` 項目（untracked），逐檔分類後處理。

---

## 硬安全閥

**禁止盲目 `git add .`** — 每個 untracked 檔案必須逐一分類，secrets 審查通過後才 commit。

HITL gate：Sonnet 出分類表 → 甲方核可 → 才執行 git add / .gitignore 修改 / commit。

---

## 完成定義

1. 所有 `??` 檔案完成分類（追蹤 / gitignore / 維持）
2. 甲方核可分類表
3. .gitignore 補齊新規則
4. `git add` 追蹤集合、`git commit`
5. `git status` 顯示 working tree clean（除已知維持項）
